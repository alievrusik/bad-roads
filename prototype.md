# bad-roads — прототип анализа жалоб на дороги

## Краткое описание

Интерактивное демо для клиента **«Томская региональная управляющая компания по ЖКХ»**: пользователь вставляет поток русскоязычных соцмедиа‑комментариев о дорогах Томска, сервер извлекает локации, оценивает тяжесть (1–10), нормализует адрес для геокодинга и отображает скопления на интерактивной карте (**Leaflet**, тайлы **CARTO** поверх данных OpenStreetMap). Дополнительный модуль принимает фотографию и запускает сегментацию по англоязычному текстовому классу.

## Предметная область

Городские дороги, эксплуатация улично‑дорожной сети, обратная связь жителей (неформализованный текст + опционально фото).

## Тип задачи

`language` + визуальная локализация (опционально): **NER / извлечение локаций**, **оценка тональности/тяжести**, **геокодирование**, **картографическая агрегация**; отдельный поток — **сегментация изображения**.

## Входы

- Сырой многострочный русский текст жалоб.
- (Опционально) изображение участка дороги + текстовый класс на английском для модели сегментации.

## Выходы

- JSON со списком точек: исходное название, нормализованный адрес, широта/долгота (если найдены), оценка 1–10, краткое резюме, частота упоминаний, уверенность, сервисные предупреждения.
- Для сегментации: нормализованный JSON с превью/оверлеем (data URL), списком полигонов/bbox, диагностикой (`processingMode`, `upstreamStatus`, …).

## Подход

1. **Текст**: вызов настроенного языкового бэкенда (`vllm` или `anthropic` в порядке `DEMO_FOUNDATION_PROVIDERS`) с жёсткой JSON‑схемой в системном промпте (английский инструктаж, русский контент). При ошибке — детерминированные эвристики по шаблонам улиц.
2. **Геоданные**: серверные запросы к **Photon** (предпочтительно для Vercel/облака), при неудаче — к **Nominatim**; контекст «Томск, Россия».
3. **Карта**: **Leaflet** + **MarkerCluster** на тайлах **CARTO light_all** (данные OSM, без прямой загрузки с `tile.openstreetmap.org`); радиус маркера масштабируется по частоте.
4. **Сегментация**: прокси к `${SAM3_API_BASE_URL}/sam3-image` с `x-api-key`, уменьшение изображений >1024 px через `sharp`, нормализация ответов и локальный визуальный fallback.

## Ограничения и риски

- Нет размеченного датасета; качество извлечения определяется промптом и конкретностью текста.
- Геокодирование может давать пропуски или смещения.
- Сегментация чувствительна к ракурсу, освещению и формулировке `text_prompt`.
- Прототип **не** подтверждает фактическое состояние покрытия в полевых условиях.
