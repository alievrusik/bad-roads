# Laplace Prototype — стандарты проекта

- **Секреты**: любые ключи и прокси‑URL фундаментальных моделей только в серверных переменных окружения; не дублировать в `NEXT_PUBLIC_*` и не логировать в клиент.
- **Вызовы моделей**: сетевые обращения к LLM / сегментации выполняются из **Route Handlers** или Server Actions, не из браузера.
- **Сегментация SAM3**: HTTP `POST` на `${SAM3_API_BASE_URL}/sam3-image` с заголовком `x-api-key`; не использовать `/segment` и не подставлять `Authorization: Bearer` для этого маршрута.
- **Текст UI**: пользовательский интерфейс и тексты помощи — на **русском**, кроме обязательных английских `text_prompt` / классов для SAM3 (или серверного RU→EN маппинга).
- **Копирайтинг**: не упоминать конкретного поставщика модели в пользовательских строках интерфейса.
- **Качество репозитория**: обязательны рабочие скрипты `npm run dev`, `npm run build`, `npm run typecheck`; при наличии — `lint` / тесты.
- **Документация**: поддерживать актуальные `README.md` (запуск, переменные окружения, smoke‑сценарий) и `prototype.md` (входы/выходы/ограничения).
- **Демо‑данные**: держать встроенный детерминированный пример кнопкой «Загрузить пример», чтобы пройти happy‑path без ручной подготовки.
