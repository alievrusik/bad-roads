# bad-roads — прототип «Карта жалоб на дороги» (Томск)

Веб‑демо для **УК Томской области**: анализ неструктурированных русскоязычных комментариев о состоянии дорог, извлечение адресов / названий улиц (для строк без координат OSM), оценка тяжести жалобы (1–10), серверное геокодирование через **Photon** (Komoot, данные OSM) с резервным **Nominatim** и визуализация на **Leaflet**. Подложка картограммы по умолчанию — стилизованная **Esri World Light Gray** (без прямой загрузки с «классического» `tile.openstreetmap.org` на клиенте); при недоступности CDN автоматический откат на тайлы **CARTO** (данные OpenStreetMap в атрибуции).

Источник демотекста при открытии страницы: **официальный API заметок OpenStreetMap** `GET https://api.openstreetmap.org/api/0.6/notes` по bbox Томска (см. `GET /api/tomsk-notes`). Если заметок в ответе меньше целевого числа строк, недостающие дополняются **синтетическими** формулировками про дороги с явным маркером `[синтетика для демо: …]` по темам публичных публикаций (например, [РИА Томск о дорожной программе и ямочном ремонте](https://www.riatomsk.ru/), [tomsk.ru — новости о дорогах](https://www.tomsk.ru/)).

Дополнительно: загрузка фото участка дороги и вызов серверной сегментации по текстовому запросу на **английском** (классы для модели), с предпросмотром и полигональным оверлеем.

Пример см. файл [`.env.example`](./.env.example).

## Переменные окружения (конфигурация фундаментальных моделей)

Секреты и эндпоинты задаются **только на сервере** (Vercel / `.env.local`). Не используйте префикс `NEXT_PUBLIC_` для ключей.

| Переменная | Назначение |
|------------|------------|
| `DEMO_FOUNDATION_PROVIDERS` | Список через запятую: `sam3`, `anthropic`, `vllm`. Для текста используются `anthropic` и `vllm` (в порядке перечисления); `sam3` не вызывается для языкового анализа. |
| `DEMO_FOUNDATION_PROVIDER` | Подсказка среды (может совпадать с приоритетным провайдером). |
| `ANTHROPIC_API_KEY` | Ключ доступа Anthropic Messages API (обязателен, если в списке провайдеров есть `anthropic`). |
| `ANTHROPIC_PROXY_URL` или `ANTHROPIC_BASE_URL` | Опционально: базовый URL совместимого шлюза (без суффикса `/v1/messages`). Если переменная **не задана**, клиент делает запрос напрямую на `https://api.anthropic.com/v1/messages`. |
| `ANTHROPIC_MODEL` | Идентификатор модели (по умолчанию `claude-sonnet-4-6`). |
| `DEMO_VLLM_BASE_URL`, `DEMO_VLLM_API_KEY`, `DEMO_VLLM_MODEL` | OpenAI‑совместимый vLLM для извлечения сущностей. |
| `SAM3_API_BASE_URL`, `SAM3_API_KEY` | Сегментация изображений (POST `${SAM3_API_BASE_URL}/sam3-image`, заголовок `x-api-key`). |

Если языковые провайдеры недоступны или не настроены, `/api/analyze` автоматически переходит на **эвристический** разбор (регулярные выражения по типовым префиксам улиц) — удобно для быстрого демо без секретов.

## Развёртывание (Vercel)

1. Подключите репозиторий и задайте переменные окружения в панели Vercel (минимум для Anthropic-сценариев: `ANTHROPIC_API_KEY`).
2. Сборка: `npm run build`, старт: `npm run start`.
3. Убедитесь, что функция `/api/analyze` имеет достаточный таймаут: маршрут объявляет `maxDuration = 60` (геокодирование идёт последовательно; для Photon — короткая пауза между запросами, при откате на Nominatim действует интервал ~1 с на обращение).

## API

### `GET /api/tomsk-notes`

Возвращает JSON `{ ok, text, stats? }`: многострочный русскоязычный блок для поля комментариев (со строками `#`-заголовками источников, они не участвуют в расчёте). Строки с координатами замечены префиксом `[[osm-note …]]`; дополнительные синтетические строки содержат хвост с `[синтетика для демо …]`. При временной недоступности upstream используется **только** синтетический набор, а тело сообщения сохраняет поясняющее предупреждение.

### `POST /api/analyze`

Тело JSON:

```json
{ "text": "многострочные русские комментарии…", "maxComments": 50 }
```

Ответ: `{ ok, warnings[], explanation?, providerUsed, items[], diagnostics }` — `items` содержат координаты (из геокодера для свободного текста и из данных OSM заметки для строк `[[osm-note …]]`), частоту и уверенность.

Текст может сочетать обычные строки со строками `[[osm-note id=… lat=… lon=…]] текст`, полученными из загрузки `GET /api/tomsk-notes`.

### `POST /api/sam3-image`

`multipart/form-data`: `image` (файл) **или** `imageUrl`, `text_prompt` (English), опционально `points_input` / `point_labels_input` / `boxes_input` как JSON‑строки.

Либо JSON с полем `image` (`data:` URL или `https://`).

## Evaluator smoke flow

1. `npm install && npm run dev`, открыть `http://localhost:3000`.
2. После загрузки страницы текстовое поле заполняется из `GET /api/tomsk-notes` (реальные OSM‑заметки + синтетическое дополнение до 100 строк при нехватке); при недоступности API отображается резерв синтетики и подсказка в интерфейсе.
3. Нажать **«Анализировать»**.
4. Ожидание: таблица с точками, JSON внизу слева, на карте справа **Esri**/резерв CARTO круглые маркеры / кластеры (точки с префиксом OSM на карте стоят точно по данным заметок). Свободные синтетические строки должны давать хотя бы частичное геокодирование улиц.
5. Альтернатива без сети OSM при отладке: **«Загрузить пример»** воспроизводит офлайновый смоук только на тексте примера.
6. При конфигурированных ключах включите языковую модель: задайте `DEMO_FOUNDATION_PROVIDERS=anthropic`, `ANTHROPIC_API_KEY` (без `ANTHROPIC_PROXY_URL` используется публичный эндпоинт Anthropic).

## Ограничения

- Точность адресов зависит от формулировок в тексте; модель не проверяет реальное состояние покрытия.
- Заметки OSM не ограничены темой «дороги» — на карте возможны офтопические отметки; синтетическое дополнение тематически окрашивается под жалобы как демоприём.
- Photon и резервный Nominatim требуют щадящего режима запросов; для длинных списков анализ может занять десятки секунд.
- Сегментация изображений зависит от качества снимка и класса в `text_prompt`.
